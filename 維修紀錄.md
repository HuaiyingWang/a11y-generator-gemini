## 問題一：Word 內嵌圖片未觸發 AI 視覺分析

### 症狀

上傳含圖片的 Word 文件後，輸出的 `<img>` 標籤出現：

```
alt="[需人工確認圖片意涵]"
```

即使圖片內容清晰（例如一張花草草原照片），仍未獲得 AI 描述。

### 根本原因

1. **Gemini 模型名稱過期**  
   `worker.js` 裡使用的 `gemini-2.5-flash-preview-09-2025` 是 preview 版，已到期失效。API 呼叫靜默失敗，catch 區塊回傳 placeholder 文字。

2. **Prompt 規則過於保守**  
   Prompt 中有一條規則：「若無法判斷圖片內容，alt 設為 `[需人工確認圖片意涵]`」。AI 遇到與文章主題（學校入學）無關的裝飾性圖片（花草）時，誤判為「無法判斷」，直接套用 placeholder。

### 修法

**`worker.js`** — 更新 Gemini 模型為 GA 穩定版：

```javascript
// 修改前（已過期的 preview 版）
const geminiUrl = `.../models/gemini-2.5-flash-preview-09-2025:generateContent?key=...`;

// 修改後（GA 穩定版）
const geminiUrl = `.../models/gemini-2.0-flash:generateContent?key=...`;
```

**`index.html`** — 改寫 AI Prompt，強制描述可見內容：

```
舊版規則（過保守）：
  若無法判斷，alt 設為 [需人工確認圖片意涵]

新版規則（明確）：
  5. 裝飾性圖片（花草、背景）仍須描述其視覺內容，
     例如：「粉紫色野花盛開的草原，陽光照耀」
  4. 只有在圖片完全空白或無法解碼時，
     alt 才設為 "[需人工確認圖片意涵]"
```

## 問題二：修完模型後仍輸出 `[AI分析失敗，請手動填寫]`

### 症狀

Cloudflare Worker 重新部署後，AI 分析回傳：

```
alt="[AI分析失敗，請手動填寫]"
```

無法判斷是 API 問題還是程式問題。

### 根本原因

原始錯誤處理只回傳通用文字，無診斷資訊，無法區分失敗原因。

### 修法

**`index.html`** — 暫時將 HTTP 狀態碼和錯誤訊息直接顯示在 alt 屬性（診斷用）：

```javascript
// 修改前
return { alt: '[AI分析失敗，請手動填寫]', caption: '' };

// 修改後（診斷用，確認修好後再移除）
if (!response.ok) {
    const errBody = await response.text();
    return { alt: `[API錯誤 HTTP${response.status}]`, caption: errBody.substring(0, 200) };
}
```

這讓使用者不需開 DevTools 就能直接在輸出 HTML 看到錯誤碼。

**相關 Commit：** `a060062` — debug: expose Gemini API error status/message in alt text for diagnosis

## 問題三：診斷後確認 HTTP 429（配額耗盡）

### 症狀

輸出後 alt 顯示：

```
alt="[API錯誤 HTTP429]"
```

### 根本原因

HTTP 429 = `RESOURCE_EXHAUSTED`（Gemini 免費額度限制）

Word 文件內嵌圖片往往是原始解析度（可能 2000×1500px 以上），轉 base64 後可達 2–5 MB。Gemini Vision API 計算 Token 時圖片越大耗費越多，一張大圖就可能耗盡免費配額觸發 rate limit。

### 修法

**`index.html`** — 新增 `resizeImageForAI()` 函式，送給 AI 前先縮圖：

```javascript
// 送給 AI 前先縮圖至 max 800px，大幅降低 Token 消耗、避免 429
const resizeImageForAI = (base64Data, mimeType, maxDim = 800) =>
  new Promise((resolve) => {
    const img = new Image();
    img.onload = () => {
      const scale = Math.min(maxDim / img.width, maxDim / img.height, 1);
      const canvas = document.createElement('canvas');
      canvas.width  = Math.round(img.width  * scale);
      canvas.height = Math.round(img.height * scale);
      canvas.getContext('2d').drawImage(img, 0, 0, canvas.width, canvas.height);
      const b64 = canvas.toDataURL('image/jpeg', 0.8)
                        .replace(/^data:image\/jpeg;base64,/, '');
      resolve({ b64, mimeType: 'image/jpeg' });
    };
    img.onerror = () => resolve({ b64: base64Data, mimeType }); // fallback 原圖
    img.src = `data:${mimeType};base64,${base64Data}`;
  });
```

套用位置：
- **Word 圖片迴圈**：`await resizeImageForAI(m[2], m[1])` — 每張 Word 內嵌圖送 AI 前先縮
- **上傳圖片迴圈**：`await resizeImageForAI(imgData.base64, imgData.mimeType)` — 一般上傳圖片同樣先縮

**重要：最終輸出 HTML 仍使用原始全解析度 data URI，縮圖只用於 AI 分析階段，不影響圖片品質。**

**相關 Commit：** `60cad28` — fix: resize images to max 800px before Gemini AI analysis to avoid HTTP 429

## 架構說明

```
使用者瀏覽器（GitHub Pages）
  ↓ POST（含 base64 縮圖）
Cloudflare Worker（a11y-gemini-proxy.love520297.workers.dev）
  ↓ 轉送 + 附加 API Key（環境變數，前端不可見）
Gemini Vision API（gemini-2.0-flash）
  ↓ 回傳 JSON { alt, caption }
使用者瀏覽器（顯示結果）
```

- Worker 限制只接受來自 `https://huaiyingwang.github.io` 的請求（Origin check）
- API Key 存在 Cloudflare 環境變數，不暴露在前端程式碼
